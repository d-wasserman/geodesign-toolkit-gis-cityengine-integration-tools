Prepare Block Associations

Title Prepare Block Associations

Summary

This tool is designed to associate all records with one geometry for the purpose of creating standard geometry for CityEngine import and then used to drive rules as part of a geodesign methodology. This tool will take in a polyline file and create CityEngine ready blocks ready for transportation and land use coordination projects.


Usage

Use this tool to create an output feature class that has simplified geometry design for the purpose of CityEngine import. This particular tool will be used to create block geometries to help with land use and transportation coordination scenarios.


Syntax

PrepareBlockAssociations (Feature_Class, Output_Feature, {Distance}, {Distance_Field}, Block_Width)

Parameter Explanation Data Type
Feature_Class Dialog Reference

The input feature classes whose features will be associated with a simplified version of the input feature class. The records will be preserved for the copy created, but the geometry will change.

___________________

Python Reference

Input feature class put into a copy features tool.

 Feature Layer
Output_Feature Dialog Reference

The resulting output feature class that has simplified geometry for importing into CityEngine. This feature class is an ouput feature class that keeps the record associated with each geometry but increases the number of feature geometries by a factor of seven (3 roads on each side to create a block on each side of a street). Keep in mind that this output will be projected into Web Mercator Auxillary Sphere in order to make the feature CityEngine ready.

___________________

Python Reference

An insert cursor is used with a newly created feature class using the input as a template. The script is close to the same for lots or streets, as the script determines treatments based on feature type. If a geometry cannot be made, the record is deleted.

Feature Class
Distance (Optional) Dialog Reference

Is the linear distance in terms of length that you want the resulting street cross section to be. Keep in mind that units are in terms of the feature classes current projection.

___________________

Python Reference

The tool uses the units of the current projection for node translations. Distances might not be entirely accurate due to changes in projection or based on various calculations.

Double
Distance_Field (Optional) Dialog Reference

If you want to reference a distance field (like Shape_Area or Shape_Length), that is in the units the same as the current projection, you can select a field here to use.

___________________

Python Reference

The tool uses the units of the current projection for node translations. Distances might not be entirely accurate due to changes in projection or based on various calculations.

Field
Block_Width Dialog Reference

This is a constant value in the current units of the used projection that is used to determine the width of the blocks created by this tool. The larger this value the longer the side streets connecting to the blocks. Be sure to check that all new segments that are not a "Main Street" have a zero street and sidewalk width.

___________________

Python Reference

A geometry dictionary is created based on the input block width value that is then unloaded to create the feature blocks.
